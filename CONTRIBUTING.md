# Contributing

This project is Phase 1 of an Interledger Protocol research project (2026–2027), built on the Open Payments stack. Contributions are welcome — especially from the [Interledger community](https://interledger.org) and builders working on the five pressure zones ([see README](./README.md#the-five-pressure-zones)).

## What kind of contributions are most useful

- **Issues that identify incorrect claims** in the docs. We would rather be corrected publicly than ship a wrong statement.
- **Pull requests that extend the demo** with additional use cases along the five pressure zones.
- **Test-network compatibility fixes** — if Interledger test network behavior changes and something breaks, a heads-up or a PR is deeply appreciated.
- **Documentation improvements** — clearer explanations for non-developer readers.

## What kind of contributions are out of scope right now

- Large architectural rewrites while Phase 1 is the reference. Phases 2–5 are where new architecture goes.
- Alternative payment rails beyond Open Payments. The whole point of this project is Open Payments specifically.

## How to contribute

1. Fork the repo.
2. Create a feature branch: `git checkout -b your-change`.
3. Commit with a clear message.
4. Open a pull request with a description of what and why.

For larger changes, please open an issue first to discuss.

## Code of conduct

Be kind. Be honest. Push back on ideas, not on people. If a claim in this repo is wrong, say so plainly and we will fix it.
